# Agent Usage Monitor

**Claude Code 사용량을 Windows 작업표시줄에서 한눈에** 보여 주는 위젯입니다.
5시간·주간 한도 사용률을 게이지로, 일·주·월 토큰과 비용(USD)을 대시보드로 실시간 확인할 수 있습니다.

- 작업표시줄 안에 **5시간 / 주간 게이지 2줄** (항상 표시)
- **대시보드** — 일/주/월 토큰·비용 차트, 모델별 비용 분해, 지금 사용 중인 프로젝트
- **한도 소진 예측** — 최근 사용 속도로 5시간 한도 도달 시각 추정
- **사용량 경고 알림** · **다크/라이트 테마** · **원클릭 자동 업데이트**

> 이 프로그램은 **이 PC에 설치·로그인된 Claude Code**의 사용량을 읽어 표시합니다.
> 사용량 수치는 Anthropic 공식 사용량 API에서, 토큰·비용 통계는 이 PC에 저장된 Claude Code 기록에서 가져옵니다.

---

## 1. 설치 전에 준비할 것

| 준비물 | 설명 |
|---|---|
| **Windows 10 / 11 (64비트)** | 별도 런타임(.NET 등) 설치가 **필요 없습니다** — 실행에 필요한 구성요소가 이미 포함되어 있습니다. |
| **Claude Code CLI (설치·로그인)** | 이 위젯은 Claude Code의 사용량을 보여 줍니다. 아직 없다면 [Claude Code 설치 안내](https://docs.anthropic.com/en/docs/claude-code)를 따라 설치하고, 터미널에서 `claude` 를 실행해 로그인해 주세요. |

> CLI가 없거나 로그아웃 상태여도 프로그램은 설치·실행되지만, 위젯에 **"⚠ Claude Code 미감지"** 가 표시됩니다.
> `claude` 로그인이 되어 있으면 자동으로 복구됩니다. (`CLAUDE_CONFIG_DIR` 환경변수로 설정 폴더를 옮긴 경우도 인식합니다.)

## 2. 설치 방법 — GitHub 릴리스에서 내려받기 (권장)

### 2-1. 릴리스 페이지 열기

아래 **최신 릴리스** 페이지로 이동합니다:

**👉 https://github.com/Weaverloft-AILabs/Agent-Usage-Monitor-Win/releases/latest**

페이지 아래쪽 **`Assets`**(자산) 목록을 펼치면 여러 파일이 보입니다.

### 2-2. 인스톨러 내려받기

**`AgentUsageMonitor-Installer.exe`** 하나만 클릭해서 내려받으면 됩니다 (약 130MB).

| 받을 파일 | 용도 |
|---|---|
| ✅ **`AgentUsageMonitor-Installer.exe`** | **이 파일 하나만 받으세요.** 설치 파일을 내부에 품고 있어 **네트워크 없이도(오프라인) 설치**됩니다. |
| ⬜ `AgentUsageMonitor-win-Portable.zip` | 무설치(포터블)로 쓰고 싶을 때만 (아래 [2-6](#2-6-무설치포터블로-쓰고-싶다면) 참고) |
| ⛔ `RELEASES`, `releases.win.json`, `*.nupkg` | **내려받지 마세요.** 앱이 자동 업데이트에 쓰는 내부 파일입니다. |

### 2-3. 인스톨러 실행

내려받은 **`AgentUsageMonitor-Installer.exe`** 를 더블클릭합니다.

- **관리자 권한이 필요 없습니다.** 사용자 폴더(`%LOCALAPPDATA%\AgentUsageMonitor`)에 설치되므로 UAC(관리자 확인) 창이 뜨지 않습니다.
- 인스톨러 창에서 **[지금 설치]** 를 누르면 **다운로드 → 보안 검사 → 설치 → 완료** 순서로 진행 상황이 표시됩니다.

### 2-4. "Windows의 PC 보호"(SmartScreen) 경고가 뜨면

이 프로그램은 아직 코드서명 인증서로 서명되지 않아, 처음 실행할 때 SmartScreen이 "인식할 수 없는 앱"으로 표시할 수 있습니다. 이때는:

> **추가 정보** 클릭 → **실행** 버튼 선택

파일은 소스가 공개된 저장소의 GitHub Actions에서 그대로 빌드되며, 빌드 과정은 저장소의 Actions 로그로 확인할 수 있습니다. (서명 인증서가 적용되면 이 경고는 사라집니다.)

### 2-5. 첫 실행

설치가 끝나고 **[시작하기]** 를 누르면, **작업표시줄 안(기본적으로 왼쪽 영역)** 에 사용량 위젯이 나타납니다.

- 위젯은 **드래그로 위치를 옮길 수 있고**, 모니터가 여러 대면 다른 모니터의 작업표시줄로도 옮길 수 있습니다.
- 처음엔 사용률이 **"로딩중…"** 으로 보이다가, 잠시 후 실제 게이지로 바뀝니다.

### 2-6. 무설치(포터블)로 쓰고 싶다면

`AgentUsageMonitor-win-Portable.zip` 을 받아 원하는 폴더에 압축을 풀고 `Agent Usage Monitor.exe` 를 실행하면 됩니다.
단, **포터블 실행에서는 자동 업데이트가 동작하지 않습니다.** 새 버전은 직접 다시 받아야 하므로, 일반 사용에는 **인스톨러 방식을 권장**합니다.

## 3. 사용법

### 작업표시줄 위젯

- **5시간 / 주간** 사용률을 게이지·퍼센트·리셋 카운트다운으로 표시합니다.
- 사용률이 경고 임계값을 넘으면 게이지 색이 바뀌고, 새 버전이 있으면 **코랄색 다운로드 배지**가 붙습니다.
- **더블클릭** → 대시보드 열기
- **우클릭** → 메뉴:
  - **새로고침** (사용량 즉시 갱신 + 새 버전 확인)
  - **대시보드 열기**
  - **표시 모드** — 작업표시줄 도킹 · 화면 위 플로팅 · 숨김
  - **설정**
  - (새 버전이 있을 때) **업데이트 설치**

### 대시보드 (위젯 더블클릭)

- 상단에 **5시간 / 주간 게이지**, 플랜 배지, **지금 사용 중인 프로젝트** 목록
- **일 / 주 / 월** 전환 — 모델별로 쌓인 **토큰 사용량 차트**, **비용 추이**, **모델별 비용 분해**, 총 토큰·총 비용
- 우상단 아이콘 — **새로고침 · 설정 · 문의하기**
- (대시보드는 열 때와 새로고침할 때의 스냅샷을 보여 줍니다. 작업표시줄 위젯은 항상 실시간입니다.)

### 설정

- **새로고침 주기** (최소 180초)
- **사용량 경고 알림** on/off 및 **경고 임계값 %** (기본 70%, 10~99% 범위) — 5시간 사용률이 임계값에 닿으면 알림을 한 번 보냅니다.
- **테마** (다크 / 라이트 / 시스템)
- **Windows 시작 시 자동 실행**
- **유휴 시 자동 업데이트** (기본 꺼짐)
- **데이터 폴더 열기**

## 4. 업데이트

설치판은 새 버전을 **자동으로 감지**합니다(주기적 확인 + 위젯 "새로고침" 시에도 확인).

- 새 버전이 있으면 위젯에 **다운로드 배지**가 표시됩니다.
- 위젯 **우클릭 → 업데이트 설치**, 또는 **설정 창**에서 버튼 한 번으로 적용됩니다. 적용 후 자동으로 재시작되며, **기존 통계·설정은 그대로 유지**됩니다.

> **주요(메이저) 버전 업그레이드**(예: 1.x → 2.x)는 설치 방식이 달라 인앱 업데이트 대신 릴리스 페이지 링크를 안내합니다.
> 이 경우 위 [설치 방법](#2-설치-방법--github-릴리스에서-내려받기-권장)대로 새 인스톨러로 설치하면 되고, 데이터·설정은 유지됩니다.

## 5. 문제 해결

### 위젯에 "⚠ Claude Code 미감지" 가 뜹니다
Claude Code CLI가 없거나 로그아웃 상태입니다. 터미널에서 `claude` 를 실행해 로그인하면 자동으로 복구됩니다.

### 설치가 느리거나 멈춘 것처럼 보입니다
1. **백신 프로그램이 검사 중** (가장 흔함) — 서명되지 않은 새 프로그램이라 V3·알약 등 백신이 실행 전 수십 초~1분 이상 잡아둘 수 있습니다. 잠시 기다리거나, 백신에서 해당 파일을 신뢰(예외)로 추가한 뒤 다시 실행해 보세요.
2. **이전 설치 잔재** — 설치가 한 번 실패한 뒤라면 `%LOCALAPPDATA%\AgentUsageMonitor` 폴더를 삭제하고 다시 설치하세요.
3. **설치 로그** — `%LOCALAPPDATA%\velopack\velopack.log` 에 남습니다. 이 파일이 **아예 없다면** 백신이 실행 자체를 막은 것이고(1번), 있다면 로그 마지막 줄이 원인을 알려 줍니다.

### 사용률(%)이 계속 "로딩중…" 이거나 갱신되지 않습니다
사용량 API 응답을 기다리는 중이거나 일시적 네트워크 문제일 수 있습니다. 위젯 **우클릭 → 새로고침** 을 눌러 보세요. 토큰·비용 통계(로컬)는 정상인데 게이지만 로딩 중이라면, 잠시 후 자동으로 채워집니다.

문제가 계속되면 대시보드/위젯의 **문의하기**로 알려 주세요 (설치 문제라면 `velopack.log` 를 함께 첨부해 주시면 도움이 됩니다).

## 6. 제거

Windows **설정 → 앱 → 설치된 앱 → Agent Usage Monitor** 에서 제거합니다.

> 제거하면 로컬 데이터(월간 통계 롤업·설정)도 **함께 삭제**됩니다.
> 통계를 남기고 싶다면 제거 전에 `%LOCALAPPDATA%\AgentUsageMonitor.Data` 폴더를 백업해 두세요.

## 7. 개인정보 안내

- 사용량 게이지는 이 PC의 Claude Code 로그인 토큰으로 **Anthropic 공식 사용량 API**를 호출해 받아옵니다. 토큰·비용 통계는 이 PC의 Claude Code 기록(`~/.claude`)을 읽어 **로컬에만** 저장합니다.
- **토큰을 갱신(회전)하지 않으며**, 이메일 등 개인정보를 저장하거나 로그로 남기지 않습니다.
- 사용량 수치와 통계를 **외부로 전송하지 않습니다.** (예외: 대시보드의 **문의하기**를 직접 사용할 때, 입력한 이메일·제목·내용만 문의 채널로 전송됩니다.)
- 비용(USD)은 공개 가격 정보를 기준으로 **추정한 값**으로, 실제 청구액과 다를 수 있습니다.
